在 LangGraph（或更广泛地说，在 Anthropic 的 Agent Skills 架构）中，**Skills 是一种通过提示词驱动、支持渐进式披露（Progressive Disclosure）的模块化能力封装机制**。它由一个标准目录结构组成，包含四个核心部分：

- `SKILL.md`：技能的元数据和使用说明（YAML + Markdown）
- `scripts/`：可执行代码（如 Python 脚本）
- `references/`：详细文档、API 说明等参考资料
- `assets/`：模板、示例文件、资源等

下面我为你写一个**完整的小 Demo：一个“生成随机笑话”的 Skill**，你可以直接运行并集成到 LangGraph Agent 中。

---

### 📁 目录结构
```
joke-generator/
├── SKILL.md
└── scripts/
    └── generate_joke.py
```

> （`references/` 和 `assets/` 可选，本例暂不使用）

---

### 1️⃣ `SKILL.md` —— 技能定义与提示词
```markdown
---
name: joke-generator
description: |
  当用户明确要求讲个笑话、需要幽默内容、想放松一下或类似请求时使用。
  此技能可以生成编程笑话、动物笑话或通用笑话。
scripts:
  - scripts/generate_joke.py
---
# 使用方法

运行 `scripts/generate_joke.py` 并传入笑话类型（可选）：
- 类型可选值：`programming`, `animal`, `general`
- 若未指定，默认生成通用笑话。

脚本会返回一个 JSON 对象：{"joke": "内容"}

## 示例调用
```bash
python scripts/generate_joke.py --type programming
\`\`\`
```


---

### 2️⃣ `scripts/generate_joke.py` —— 可执行逻辑
```python
#!/usr/bin/env python3
import json
import random
import argparse

JOKES = {
    "programming": [
        "为什么程序员分不清万圣节和圣诞节？因为 Oct 31 == Dec 25！",
        "有一个 null 指针走进了酒吧... 然后它就崩溃了。",
    ],
    "animal": [
        "为什么鸡过马路？为了证明它不是懦夫！",
        "企鹅去银行存钱，柜员问：‘有账户吗？’ 企鹅说：‘没有，我是企鹅。’"
    ],
    "general": [
        "我昨天把手机扔进海里了——现在它变成了‘盐水机’。",
        "我妈说我小时候很聪明，直到我把遥控器放进微波炉‘充电’。"
    ]
}

def main():
    parser = argparse.ArgumentParser()
    parser.add_argument("--type", default="general", choices=["programming", "animal", "general"])
    args = parser.parse_args()

    joke = random.choice(JOKES[args.type])
    print(json.dumps({"joke": joke}, ensure_ascii=False))

if __name__ == "__main__":
    main()
```

### ✅ 如何在 LangGraph 中使用这个 Skill？

虽然 LangGraph 本身不直接解析 `SKILL.md`，但你可以：

1. **在 Agent 的系统提示中引用该 Skill 的描述**（L1 元数据）：
   ```text
   你拥有以下技能：
   - joke-generator: 当用户要求讲个笑话、需要幽默内容时使用。
   ```

2. **当 LLM 决定调用该技能时，Agent 执行对应脚本**（模拟工具调用）：
   ```python
   import subprocess
   result = subprocess.run(
       ["python", "joke-generator/scripts/generate_joke.py", "--type", "programming"],
       capture_output=True, text=True
   )
   joke_data = json.loads(result.stdout)
   ```

3. **将结果返回给 LLM 继续对话**。

---

### 🔍 渐进式披露（Progressive Disclosure）体现

- **L1（启动时加载）**：所有 Skill 的 `name` + `description` → 用于路由决策
- **L2（调用时加载）**：完整的 `SKILL.md` 正文 → 帮助 LLM 精确理解如何调用
- **L3（按需加载）**：`scripts/`, `references/`, `assets/` → 实际执行或查资料

这样避免了一次性把所有技能细节塞进上下文，节省 token。

是 Python 标准库 `argparse` 模块的核心用法，**目的是让脚本能够从命令行接收参数**。下面我来逐行详细解释其作用和必要性。

---

### 🔍 为什么需要 `argparse`？

在 LangGraph 或多智能体系统中，**技能（Skill）通常以独立脚本的形式存在**，并由主 Agent **通过命令行调用**。例如：

```bash
python scripts/generate_joke.py --type programming
```

为了让 `generate_joke.py` 能“听懂”用户传入的 `--type programming` 这个指令，就需要一个工具来解析命令行参数。`argparse` 就是 Python 官方推荐的、用于处理命令行参数的标准库。

---

### 📌 逐行详解

#### 1. `parser = argparse.ArgumentParser()`
- **作用**：创建一个参数解析器对象。
- **类比**：就像你请了一个“翻译官”，专门负责把用户在终端输入的命令“翻译”成 Python 能理解的变量。

#### 2. `parser.add_argument("--type", default="general", choices=["programming", "animal", "general"])`
- **作用**：定义脚本接受的一个**可选参数**（因为以 `--` 开头）。
- **参数详解**：
  - `"--type"`：参数名。用户必须用 `--type 编程类型` 的形式传参。
  - `default="general"`：如果用户**没有提供 `--type`**，就默认使用 `"general"`。
  - `choices=[...]`：**限制合法输入值**。如果用户输入了不在列表中的值（比如 `--type politics`），程序会自动报错并退出，避免无效调用。
    ```bash
    # 合法
    python generate_joke.py --type animal
    
    # 非法 → 报错
    python generate_joke.py --type politics
    ```

#### 3. `args = parser.parse_args()`
- **作用**：**真正执行解析**，将命令行输入转换为一个 Python 对象（`Namespace`）。
- **结果示例**：
  - 如果运行 `python generate_joke.py --type programming`  
    → `args.type == "programming"`
  - 如果运行 `python generate_joke.py`（不带参数）  
    → `args.type == "general"`（因为有默认值）

之后你就可以用 `args.type` 来获取用户想要的笑话类型了！

---

### ✅ 为什么这对 Skill 架构至关重要？

1. **标准化接口**：所有 Skill 脚本都通过命令行参数交互，Agent 无需知道脚本内部逻辑，只需按约定传参即可。
2. **安全性**：`choices` 防止 LLM 传入恶意或无效参数（如 `--type ../../../etc/passwd`）。
3. **灵活性**：未来可以轻松扩展新参数（比如 `--language zh`）。
4. **可测试性**：你可以直接在终端手动测试脚本，无需启动整个 Agent 系统。

---

### 💡 小结

| 代码 | 作用 |
|------|------|
| `ArgumentParser()` | 创建命令行参数“翻译官” |
| `add_argument(...)` | 告诉翻译官：“我只接受 `--type`，且只能是那三种值，没给就用 `general`” |
| `parse_args()` | 让翻译官开始工作，把用户输入变成 `args.type` 变量 |

这样，你的 Skill 脚本就能**安全、可靠、标准化地**被 LangGraph Agent 调用了！
