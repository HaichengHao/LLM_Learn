# 补充之pretty_print()

------------
`pretty_print()` 是 LangChain（以及 LangGraph）中用于**格式化并打印消息内容**的一个便捷方法，主要用于在终端或控制台中**以更易读的方式展示消息**（如 AI 回复、用户输入等）。

### 具体作用：
- 它会将 `AIMessage`、`HumanMessage`、`SystemMessage` 等 LangChain 消息对象的内容（包括文本、工具调用、附加信息等）**结构化地输出到控制台**。
- 对于普通文本回复，它通常只打印出 `.content` 字段；
- 如果消息包含 **工具调用（tool calls）**、**中间步骤** 或 **元数据**，也会一并美观地展示出来。

---

### 举个例子：

假设你有如下 AI 回复消息对象：

```python
from langchain_core.messages import AIMessage

msg = AIMessage(content="你好，张三！你喜欢足球。")
```

直接 `print(msg)` 可能输出：

```
AIMessage(content='你好，张三！你喜欢足球。', additional_kwargs={}, response_metadata={})
```

而调用：

```python
msg.pretty_print()
```

会输出更干净的：

```
你好，张三！你喜欢足球。
```

或者在 Jupyter Notebook 中，可能还会带颜色高亮（如 AI 回复用蓝色，用户输入用绿色等）。

---

### 在你的代码中：

```python
msg1["messages"][-1].pretty_print()
```

意思是：**取出本轮对话中最新的一条消息（通常是 AI 的回复），然后美观地打印出来**，避免看到冗长的对象表示。

> ✅ 总结：`pretty_print()` 就是一个“美化打印”函数，方便调试和查看对话内容，**不影响程序逻辑**，只用于输出展示。

如果你不需要格式化输出，也可以直接用：
```python
print(msg1["messages"][-1].content)
```

# checkpointer

值得注意的是,之前在langchain中标定同一个历史数据指定的是session_id  
而在langgraph中需要指定的是thread_id

对比
- langchain
```python
	config=RunnableConfig(
		configurable={
			'session_id':'your_session_id'
		}
    )
```
- langgraph
```python
	config=RunnableConfig(
		configurable={
			'thread_id':'your_session_id'
		}
    )
```
## 内存检查点  

>这里不细说了,生产中几乎不用它

*写法*  
- 先导入 `from langgraph.checkpoint.memory import InMemorySaver`
- 然后在编译图的时候传入`builder.compile(checkointer=InMemorySaver())`
- 编写好config,调用的时候传入thread_id

## 数据库检查点之sqlite

*写法*  
- 先导入 `from langgraph.checkpoint.sqlite import SqliteSaver import sqlite3`
- 构造连接对象
```python
    conn = sqlite3.connect(
	database="./sqlite_data.db", 
	check_same_thread=False) #tips:可以看一下，这样得到的是一个Connection对象
	sqliteDB = SqliteSaver(conn=conn) #important:SqliteSaver要求传入一个Connection对象

```
- 编译图的时候指定checkpointer
```python
 graph = (
        StateGraph(DemoState)
        .add_node('node_1',node_1)
        .add_edge(START,'node_1')
        .add_edge('node_1',END)

        .compile(checkpointer=sqliteDB)
    )
```


## 数据库检查点之posrgres
*写法*  
- 先导入 `from langgraph.checkpoint.postgres import PostgresSaver import psycopg`
- 构造连接对象
```python
    conn=psycopg.connect(
        dbname="llmdb",
        autocommit=True
    )

    psqlDB=PostgresSaver(conn=conn)
    psqlDB.setup()#important: 首次运行时创建 checkpoint 表
```
- 编译图的时候指定checkpointer
```python
 graph = (
        StateGraph(DemoState)
        .add_node('node_1',node_1)
        .add_edge(START,'node_1')
        .add_edge('node_1',END)

        .compile(checkpointer=psqlDB)
    )
```

## Agent的记忆实现

其实就是创建agent的时候指定checkpointer就行,构造checkpointer的方式就是上面那几种,当然还有像用redis等构建的,大同小异

```python
from langchain.chat_models import init_chat_model
from langgraph.checkpoint.memory import InMemorySaver
from langgraph.checkpoint.postgres import PostgresSaver
from langchain.agents import create_agent
from dotenv import load_dotenv
import os
import psycopg
load_dotenv()

zai_key = os.getenv('zhipu_key')
zai_url=os.getenv('zhipu_base_url')

# ==========定义大模型 ==========
llm = init_chat_model(
    model="glm-4",
    model_provider="openai",
    api_key=zai_key,
    temperature=0.0,
    base_url=zai_url
)

# 定义短期记忆使用内存（生产可以换 RedisSaver/PostgresSaver）
# checkpointer = InMemorySaver()

#tips:使用psql作检查点
psqlcheckpointer = PostgresSaver(
    conn=psycopg.connect(
        autocommit=True,
        dbname="lg_p1",
    )
)
psqlcheckpointer.setup()

# agent = create_agent(model=llm,checkpointer=checkpointer)
agent = create_agent(model=llm,checkpointer=psqlcheckpointer)
# 多轮对话配置，同一 thread_id 即同一会话
config = {"configurable": {"thread_id": "user-001"}}

msg1 = agent.invoke({"messages": [("user", "你好，我叫张三，喜欢足球，60字内简洁回复")]}, config)
print(msg1)
msg1["messages"][-1].pretty_print()

# 6. 第二轮（继续同一 thread）
msg2 = agent.invoke({"messages": [("user", "我叫什么？我喜欢做什么？")]}, config)
msg2["messages"][-1].pretty_print() #
```
