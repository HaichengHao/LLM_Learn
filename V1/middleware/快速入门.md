在 LangChain 1.x 版本中，中间件（Middleware）是一个**可插拔的拦截器系统**，它允许开发者在智能体（Agent）执行流程的关键节点插入自定义逻辑，从而实现对 Agent 行为的精细化控制。

简单来说，你可以把它想象成一个“洋葱模型”的管道：请求会依次穿过一层层中间件，到达核心的 Agent 逻辑，然后响应再按相反的顺序穿回来。这使得你可以在模型调用前后、工具执行前后等环节，灵活地添加各种功能。

### 🤔 为什么需要中间件？

在没有中间件的时代，Agent 的运行过程像一个“黑箱”，开发者难以干预其内部决策和工具调用，一旦出现问题（如成本失控、调用敏感工具、陷入死循环）就很难定位和控制。

中间件的出现正是为了解决这些问题，它的核心价值在于：

*   **提升可控性**：可以干预模型何时调用工具、如何生成提示词。
*   **增强安全性**：对敏感操作（如发送邮件、修改数据库）进行审核，对隐私数据进行脱敏。
*   **优化资源与成本**：防止 Token 超限、重复调用、死循环，实现成本控制。
*   **提高可靠性**：方便地实现日志记录、监控、重试、降级和缓存等机制。

### 🛠️ 中间件能用来做什么？

中间件通过一系列“钩子”（Hooks）来工作，这些钩子是 Agent 生命周期中的特定触发点。你可以在这些钩子中插入代码，实现各种功能。

以下是几种核心的钩子类型及其典型应用场景：

| 钩子类型 | 触发时机 | 典型应用场景 |
| :--- | :--- | :--- |
| **`@before_model`** | 模型调用前 | 动态修改提示词、输入验证、权限检查 |
| **`@after_model`** | 模型调用后 | 输出内容安全审查、结果格式化 |
| **`@wrap_model_call`** | 包裹整个模型调用 | 实现缓存、重试、动态模型切换（降级） |
| **`@wrap_tool_call`** | 包裹整个工具调用 | 工具权限校验、参数修改、错误处理 |
| **`@before_tool`** | 工具执行前 | 记录工具使用日志、检查调用权限 |
| **`@after_tool`** | 工具执行后 | 对工具返回结果进行后处理或验证 |

### 🧩 如何使用中间件？

在 LangChain 1.x 中，创建 Agent 的标准方式是使用 `create_agent` 函数，并通过 `middleware` 参数传入一个中间件列表。

```python
from langchain.agents import create_agent
from langchain.agents.middleware import SummarizationMiddleware

# 将中间件实例传入 create_agent
agent = create_agent(
    model="gpt-4o",
    tools=[...],
    middleware=[
        SummarizationMiddleware(...) # 例如：自动总结过长的对话历史
    ]
)
```

LangChain 1.x 不仅提供了一套声明式的 API 来让你轻松创建自定义中间件，还内置了多种开箱即用的预构建中间件，覆盖了最常见的生产需求：

*   **`SummarizationMiddleware`**：当对话历史过长时，自动进行总结压缩，以节省 Token。
*   **`HumanInTheLoopMiddleware`**：在执行敏感工具（如发送邮件）前暂停，等待人工审批。
*   **`PIIMiddleware`**：自动识别并脱敏输入中的个人身份信息（如邮箱、电话）。
*   **`ModelCallLimitMiddleware`**：限制模型的最大调用次数，防止成本失控。
